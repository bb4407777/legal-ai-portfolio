# 自动化工作流引擎

Lawlink 的核心差异化能力之一，是它不只是案件管理数据库，更是一个**自动化工作流引擎**。以下是几个关键流程。

---

## 1. 短信（12368）→ 待办任务 → 日历

**痛点：** 法院通过 12368 发送开庭通知、缴费通知、审限提醒等短信。大部分律师手动阅读 → 记日历 → 设提醒，容易遗漏。

**自动化流程：**

```
SMS 接收
   │
   ▼
文本解析（提取案号/法院/时间/事项）
   │
   ▼
匹配对应案件（通过案号自动关联）
   │
   ▼
创建待办任务（Task）
   │
   ├── 开庭通知 → 自动写入系统日历（Apple Calendar）
   └── 其他通知 → 按类型分类挂入案件时间线
```

**效果：** 零人工干预，从短信到日历全自动。

## 2. 开庭信息 → 自动同步日历

**流程：**

```
开庭排期（系统内/短信解析）
       │
       ▼
自动提取：时间、地点、法庭、法官、案号
       │
       ▼
写入 Apple 日历（标题 + 时间 + 备注）
       │
       ▼
案件时间线自动追加开庭记录
```

## 3. OCR 证据管道

**痛点：** 微信聊天记录作为证据使用时，需要截屏 → 排版 → 生成证据清单。手动操作极其耗时。

**自动化流程：**

```
微信聊天录屏（MP4）
       │
       ▼
ffmpeg 逐帧截图
       │
       ▼
ocr-vision 文本识别
（区分左右气泡：左=对方，右=我方）
       │
       ▼
结构化提取关键信息
       │
       ▼
自动生成证据清单（PDF）
       │
       ▼
关联案件文档
```

## 4. 关键期限自动提醒

**流程：**

```
案件关键日期
  ├── 上诉期（判决送达后 15 天）
  ├── 举证期
  ├── 保全到期
  ├── 续封到期
  └── 执行时效
  
       │
       ▼
到期前 N 天自动创建 Task
       │
       ▼
CLI 脚本每日巡检
       │
       ▼
超期未处理 → 标记提醒
```

## 5. 案件生命周期自动推进

通过状态机设计，案件在以下节点可自动推进：

- 收案 → 自动生成所内案号
- 立案 → 自动创建立案材料文件夹
- 开庭 → 自动更新案件状态为「已排期」
- 判决 → 自动开始上诉期倒计时
- 归档 → 自动触发归档检查清单

---

这些工作流无一例外来源于**真实办案中的重复劳动**。每一个自动化节点，对应一个曾经手动操作中犯过的错、漏过的期限。
